# 3-Taps! 仕様書（初版）

## 1. 文書情報
- 文書名: 3-Taps! 仕様書
- 対象アプリ: `task-manager-nuxt`
- 版: v0.1（現状実装ベース）
- 作成日: 2026-02-10
- 想定利用者: 開発者、レビュー担当、ポートフォリオ閲覧者

## 2. アプリ概要
3-Taps! は、日付選択 -> 科目選択 -> 種別選択（課題/試験/補講）の 3 タップで予定を登録できる学習タスク管理アプリ。

主目的:
- 学習予定の登録と日付ベース管理
- 直近予定の把握
- 科目ごとの予定追加を簡潔にする

## 3. 対象範囲
本仕様書は、以下の画面と機能を対象とする。
- ホーム画面（`/`）
- 友だち画面（`/friends`）
- 共通ナビゲーション
- ブラウザ `localStorage` 永続化

## 4. 画面仕様

### 4.1 ホーム画面（`/`）
主要セクション:
- ヘッダー
- 直近の予定カード
- カレンダーカード
- 選択日の予定カード
- 科目追加セクション
- 各種モーダル/オーバーレイ
- 下部ナビ（ホーム/友だち）

機能:
- 予定一覧（直近）
- 当日強調表示（直近カード内）
- 月移動可能なカレンダー
- 曜日ヘッダー（日〜土）
- カレンダー上の予定種別ドット表示
- 日付選択
- 科目選択 -> 種別バブル -> 予定追加
- 予定削除
- 科目追加（名前+絵文字）
- 科目削除
- 予定メモ追加/更新/削除（空文字保存で削除）
- 共有パネルUI（現在は停止中表示）

表示レイアウト:
- モバイル基本: 1カラム
- PC幅（`min-width: 900px`）: 2カラム
- 左列: 直近の予定、カレンダー
- 右列: 選択日の予定、科目セクション

### 4.2 友だち画面（`/friends`）
主要セクション:
- ヘッダー
- 友だちの直近予定カード
- 共有コード入力欄
- 下部ナビ（ホーム/友だち）

機能:
- 共有コード入力UI
- 接続ボタンUI
- 共有予定一覧UI

現状:

### 4.3 共通画面遷移
- ルート遷移に `page` トランジション（`out-in`）を適用
- アニメーション: opacity + scale（200ms）

## 5. 操作仕様（ホーム）

### 5.1 予定追加
操作手順:
1. カレンダーで日付を選択
2. 科目ボタンをタップ
3. 種別バブルから `課題` / `試験` / `補講` を選択

動作:
- 選択日付に予定を1件追加
- 追加後、種別バブルを閉じる
- 共有有効時のみ共有同期処理を試行（現状無効）

### 5.2 予定削除
操作:
- 直近予定カード内の「削除」ボタン

動作:
- 確認ダイアログ表示
- 承認時のみ削除

### 5.3 予定メモ
操作:
- 予定行をタップしてメモモーダルを開く
- メモ編集後「保存」

表示:
- メモ付き予定には「詳細あり」バッジを表示

### 5.4 科目追加
操作:
- 「＋」ボタン -> 科目追加モーダル
- 科目名入力
- アイコン選択
- 「追加」

### 5.5 科目削除
操作:
- 「科目を管理」 -> 管理モーダル -> 削除

動作:
- 確認ダイアログ表示
- 承認時のみ科目マスタから削除
- 既存予定は残る（科目名解決不可時は「（削除済み）」表示）

### 5.6 カレンダー
仕様:
- 月切替ボタン（前月/次月）
- 月変更時、選択日が月末を超える場合は当月最終日に補正
- 当日は水色枠
- 選択日はオレンジ系強調
- 当日かつ選択日は2重表現
- 予定種別ドット:
- `課題`: 青
- `試験`: 赤
- `補講`: 黄

## 6. データ仕様

### 6.1 Task
```ts
type Task = {
  id: string
  date: string // YYYY-MM-DD
  subjectId: string
  type: string // 課題 | 試験 | 補講
  memo?: string
}
```

### 6.2 Subject
```ts
type Subject = {
  id: string
  name: string
  emoji: string
}
```

### 6.3 SharedTask（友だち画面側）
```ts
type SharedTask = {
  id: string
  date: string
  subject: string
  type: string
}
```

## 7. 永続化仕様

保存先:
- ブラウザ `localStorage`

キー:
- `task-manager-tasks`（予定配列）
- `task-manager-subjects`（科目配列）
- `shared-tasks-code`（共有コード）
- `task-manager-shared-ids`（共有済みID管理）

保存タイミング:
- `tasks` / `subjects` 変更時に深い監視
- 300ms デバウンス後に保存

起動時:
- 既存データを読み込み
- 破損JSONは `console.warn` 出力後、該当キーを削除

## 9. 非機能要件（現状）

### 9.1 パフォーマンス/UX
- 予定リストはカード内スクロール
- 高さ固定でレイアウトジャンプを抑制
- 上段/下段リストの高さ:
- 直近: 150px
- 選択日: 80px

### 9.2 レスポンシブ
- モバイル優先
- `>=520px` でカード外観強調（角丸・影）
- `>=900px` で2カラム

### 9.3 アクセシビリティ
- 主要ボタンに `aria-label` 付与（一部）
- モーダルは背景クリックで閉じる

## 10. 制約・注意事項
- データはローカルブラウザ保存のみ
- 端末・ブラウザ変更時にデータは引き継がれない
- ログイン機能なし
- 絵文字表示は端末フォントに依存
- 共有機能は停止中（UIのみ）
